# OFM004 - Ambiguous wikilink target

**Severity:** error
**Fixable:** no
**Added in:** Phase 4

## What it does

Errors when a wikilink target matches more than one `.md` file in the vault.
In default `path-relative` mode this most often means a bare basename such as
`[[index]]` matches multiple files. In `obsidian-fuzzy` mode it can also mean a
path-like suffix such as `[[sources/foo]]` matches multiple trailing paths.
The resolver cannot decide which file to link to, so the correct fix is to
spell out a more specific relative path.

## Bad example

Vault contains `notes/index.md` and `archive/index.md`:

```markdown
See [[index]] for details.
```

## Good example

```markdown
See [[notes/index]] for details.
```

## Config

OFM004 takes no options.
