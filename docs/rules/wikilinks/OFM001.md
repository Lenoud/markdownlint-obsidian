# OFM001 - Broken wikilink

**Severity:** error
**Fixable:** no
**Added in:** Phase 4

## What it does

Resolves every `[[wikilink]]` target against the vault file index. Fires an
error whenever the target cannot be found under any of Obsidian's three
default resolution strategies (exact path, case-insensitive path, basename).
When `wikilinks.resolveMode` is set to `obsidian-fuzzy`, path-like targets also
try a path-suffix step before basename matching. For example,
`[[sources/foo]]` can resolve to `wiki/sources/foo.md`; bare links like
`[[foo]]` still use basename matching. Embed wikilinks (`![[...]]`) are ignored
— the embed rule family (OFM020+) owns that territory.

## Config

Disable this rule in `.obsidian-linter.jsonc`:

```jsonc
{
  "rules": {
    "OFM001": { "enabled": false }
  }
}
```

You can also suppress the entire wikilink resolution pipeline with
`"resolve": false` or the `--no-resolve` CLI flag.

Mixed-namespace vaults can opt into Obsidian-style suffix resolution:

```jsonc
{
  "wikilinks": {
    "caseSensitive": false,
    "allowAlias": true,
    "resolveMode": "obsidian-fuzzy"
  }
}
```
