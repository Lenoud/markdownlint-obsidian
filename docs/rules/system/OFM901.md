# OFM901 - Internal parser error

**Severity:** error
**Fixable:** no
**Added in:** Phase 1

## What it does

Reports an unexpected tool failure while linting a file. OFM901 is a fallback
diagnostic: the linter could not classify the exception as a known rule,
frontmatter parse error, custom-rule load failure, or fix conflict.

## What should not produce OFM901

Regular markdownlint violations should surface as their original `MDxxx` rule
codes. For example, MD012 uses an upstream fix sentinel,
`fixInfo.deleteCount = -1`, for whole-line deletion. Because that fix cannot be
represented by the `markdownlint-obsidian` column-based fix model, the adapter
omits the fix and reports MD012 without autofix metadata. It should not surface
as `OFM901: Fix.deleteCount must be >= 0`.

## What to do

If OFM901 appears, capture:

- the input file path,
- the full diagnostic text,
- the active config file,
- the package version,
- and a minimal file excerpt when possible.

Then file an issue with those details so the fallback can be replaced by a
specific rule or adapter behavior.
